# Менеджер заметок на Go
## Инструкция по запуску проекта
### 1. Запуск проекта
Для запуска проекта выполните в терминале следующую команду:
```bash
go run app/main.go
```
### 2. Вывод пояснений через флаг
Можно добавить флаги:
* `go run app/main.go --info` &mdash; флаг для вывода информации о проекте
* `go run app/main.go --help` &mdash; флаг для вывода инструкции по использованию
* `go run app/main.go --file=file.json` &mdash; флаг для выбора файла хранения заметок

## Описание проекта
### 1. Структура приложения

* **`app/`**
  * `main.go` — **точка входа**. Инициализирует конфигурацию, собирает зависимости и связывает слои между собой для запуска интерфейса.
* **`config/`**
  * `config.go` — **слой конфигурации**. Отвечает за хранение и чтение глобальных настроек приложения.
  * `config_test.go` — **unit-тесты**. Проверка корректности инициализации глобальных переменных.
* **`models/`**
  * `types.go` — **модели данных**. Содержит описание структуры заметки `Note` и её полей.
* **`notes/`** — основной функциональный модуль приложения, разделенный на следующие слои:
  * `interface.go` — **слой работы с CLI**. Отвечает за отображение интерактивного текстового меню в терминале, чтение команд пользователя, форматирование и вывод данных.
  * `interface_test.go` — **unit-тесты**. Проверка консольного интерфейса.
  * `service.go` — **слой бизнес-логики**. Содержит правила валидации входных данных и фильтрацию заметок по тегам.
  * `service_test.go` — **unit-тесты**. Проверка бизнес-логики.
  * `repo.go` — **абстрактный интерфейс для работы с данными**. Объявляет интерфейс `Repository`, благодаря которому бизнес-логика не зависит от реализации хранилища.
  * `storage.go` — **слой хранения**. Реализует структуру `Store`, которая загружает сохраненные JSON-данные в память при старте и обеспечивает безопасную работу с ними в многопоточной среде, перезаписывая файл на диске при изменениях.
  * `storage_test.go` — **unit-тесты**. Проверка устойчивости приложения к некорректному пользовательскому вводу.

## 2. Использование возможностей Go
В проекте применены ключевые возможности языка Go, изученные на курсе:

1.  **Разделение кода на пакеты**: Код логически разделён на независимые пакеты, что полностью исключает циклические зависимости.
2.  **Go Modules и файл `go.mod`**: Приложение инициализировано как независимый Go-модуль. Управление внутренними зависимостями и путями импорта пакетов осуществляется через абсолютные пути.
3.  **Использование встроенных структур данных**:
    *   `struct` — для моделирования сущностей
    *   `slice` — для динамического управления списками заметок и массивами тегов
4.  **Управление потоками**: В слое хранения `storage.go` применен мьютекс `sync.Mutex`. Это гарантирует раздельный доступ горутин к общим данным и изолирует операции внутри методов хранилища.
5.  **Обработка ошибок**: В приложении реализована обработка ошибок на всех этапах работы приложения.
7.  **Unit-тестирование**: Для проверки корректности работы написаны модульные тесты с использованием встроенного пакета `testing`.
